1 Push Code to GitHub

Make sure your code is in GitHub and latest changes are pushed.

  1. Open Terminal/Command Prompt
  2. Navigate to your foodrescue folder
  3. Run:
    git add .
  4. Run:
    git commit -m "Ready for deployment"
  5. Run:
    git push origin main

‚úì Done! Your code is now on GitHub.

2 Deploy Backend to Railway

Railway will host your Node.js server + PostgreSQL database.

  1. Go to https://railway.app
  2. Click "Start Project"
  3. Sign up with GitHub
  4. Click "New Project"
  5. Select "Deploy from GitHub"
  6. Find your foodrescue repo
  7. Click "Deploy"
‚úì Railway auto-detects Node.js and starts building!

Add PostgreSQL Database:

  1. In Railway, click "+ Add"
  2. Select "Database" ‚Üí "PostgreSQL"
  3. Click "Create"
‚úì PostgreSQL is created! Railway sets DATABASE_URL automatically.

Set Environment Variables:

  1. Click the Node.js service
  2. Click "Variables" tab
  3. Add these:
    JWT_SECRET=[generate below]
    SESSION_SECRET=[generate below]
    NODE_ENV=production
    PORT=3000
‚ö†Ô∏è Generate Random Secrets:
Go to your computer terminal and run:
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
Run this TWICE - use the output for JWT_SECRET and SESSION_SECRET separately.

Get Your Backend URL:

  1. In Railway, click the Node.js service
  2. Look for "Public URL" or "Domain"
  3. It looks like: https://your-app-xyz.railway.app
  4. Copy this URL - you'll need it for Vercel!

‚úì Done! Your backend is now live on Railway.

3 Deploy Frontend to Vercel

Vercel hosts your React app and automatically connects to your Railway backend.

  1. Go to https://vercel.com
  2. Click "Sign Up"
  3. Sign up with GitHub
  4. Click "Add New..." ‚Üí "Project"
  5. Click "Import Git Repository"
  6. Find your foodrescue repo
  7. Click "Import"

Configure Build Settings:

  1. Framework Preset: Leave blank
  2. Build Command:
    npm run build
  3. Output Directory:
    dist
  4. Install Command:
    npm install

Set Environment Variables (IMPORTANT!):

  1. In Vercel, click "Environment Variables"
  2. Add this variable:
    VITE_API_URL=https://your-app-xyz.railway.app
    Replace "your-app-xyz" with your actual Railway backend URL from Step 2!
  3. Click "Save"
‚ö†Ô∏è Important: Use your ACTUAL Railway URL from Step 2, not the example URL!

Deploy:

  1. Click "Deploy"
  2. Wait 2-3 minutes for build
  3. Once done, you'll see "Visit" button
  4. Click "Visit" to see your live app!

‚úì Done! Your frontend is now live on Vercel.

‚ö†Ô∏è Troubleshooting

Problem: Login fails with "Connection error"

  1. Go to Vercel dashboard
  2. Find your project
  3. Go to "Environment Variables"
  4. Check if VITE_API_URL is set to your Railway URL
  5. If not, add it and redeploy

Problem: App is blank/white screen

  1. Press F12 to open DevTools
  2. Go to "Console" tab
  3. Look for red error messages
  4. Check Vercel deployment logs

Problem: Database connection error on Railway

  1. Go to Railway PostgreSQL service
  2. Check "Connect" tab to verify DATABASE_URL is set
  3. Make sure Node.js service has DATABASE_URL in variables
